想法

想法也就是你平时读书的时候在书中发表的想法。当你将前后缀设置为两个星号(**)时，导出的想法在 Markdown 中将会被加粗。

代码块

当你借助插件复制书本中的代码块的时候，你可以在这里设置代码块前后缀。

比如假设书中有一段代码，当你将代码块前缀设置为```python同时将代码块后缀设置为```时，你复制得到的文本在 Markdown 中会按照 python 的语法高亮显示。

有时你可能不希望导出的代码块前后包含任何内容，这时你可以将代码块前后缀设置为空。

正则匹配

当微信读书中的三种标注不能满足你添加前后缀的需求时，你可以借助正则表达式匹配标注，被匹配的标注可被添加前后缀。比如，当你要匹配一个开头为"提示"二字的标注时，可输入正则表达式：

^提示[\S ]*$

下面是一些比较常用的正则表达式：

  • 匹配开头为"步骤x："(x为0到99中任意整数)的标注：^步骤[0-9]{1,2}[:：][\S ]*$
  • 匹配图片标题如"图2-5 404错误响应"：^图[0-9]{1,2}-[0-9]{1,2} [\S ]*$
  • 匹配表格标题如"表1-1 Flask的依赖包"：^表[0-9]{1,2}-[0-9]{1,2} [\S ]*$
  • 匹配开头为数字加一点的标注如"1．Flask与HTTP"：^[1-9](\.|\. |．)[\S ]*$
  • 匹配开头为"注意："或"附注："或"提示："的标注如"提示：你可以借助微信读书笔记助手快速导出笔记为 Markdown 文本"：^(注意|附注|提示)[:：][\S ]*$

注意：

  • 选中正则表达式右方的选中框该正则表达式才会生效。
  • 因为正则表达式的设置很容易影响导出效果，所以最好在对正则表达式进行充分测试后再使用正则匹配。
其他

选中该选项之后，标注及纯文本书评中的 Markdown 特殊字符（如 \ * { [ ( + < _ ` !）将被转义。

一个常见的应用是，当你的标注中包含星号(*)时，为防止其导致生成 Markdown 斜体或黑体，可开启该选项以转义这个字符。

选中该选项之后，导出的标注（本章标注或是全书标注）中将包含你在书中留下的想法。

选中该选项之后，导出的热门标注中将包含每条热门标注的标记人数。
